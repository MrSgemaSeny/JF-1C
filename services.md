# Архитектура и реализация проекта: "Финансовый Сервис"

Этот документ содержит полную спецификацию архитектуры, структуру проекта на основе FSD (Feature-Sliced Design) и руководство по реализации MVP для финансового портала.

---

## 1. Концепция продукта
**Продукт**: Экосистема для малого и среднего бизнеса (Казахстан).
**Ключевой стек**:
* **Frontend**: React 18 + Vite + TypeScript.
* **Backend**: Java 21 + Spring Boot 3.x (Модульный монолит).
* **База данных**: PostgreSQL 16 (с поддержкой JSONB для гибких отчетов).
* **Инфраструктура**: Docker, MinIO (S3) для хранения документов и защищенной раздачи видео.

---

## 2. Архитектура FSD (Frontend)

Для достижения "Product Ready" состояния мы используем прагматичный подход FSD.

### Структура папок
```text
src/
  app/              # Инициализация приложения, роутинг, глобальные провайдеры
  pages/            # Сборка страниц из виджетов (home, services, crm, etc.)
  widgets/          # Крупные блоки (Header, Hero, ServiceList, CRM-Table)
  features/         # Бизнес-логика (AddTaskForm, CalculateTax, AuthLink)
  entities/         # Сущности (Task, Client, Service). Интерфейсы + DTO
  shared/           # Общие компоненты (UI-kit, API-клиент, Lib)

```

### Правила импортов

1. Запрещены импорты "снизу-вверх" (например, `entities` не может знать о `features`).
2. Каждый слой обязан иметь `index.ts`, который является единственной точкой доступа к публичному API модуля.

---

## 3. Описание ключевых компонентов

### А. Главный экран (Hero)

Использует анимации `framer-motion` и декоративный Canvas-фон.

* **Слой**: `widgets/hero`
* **Ключевые фичи**: Адаптивная сетка `grid-cols-12`, динамическая типографика, акцентные плашки через CSS-позиционирование.

### Б. Витрина услуг (Services)

* **Слой**: `widgets/services-list`
* **Концепция**: "Bento Grid". Модульные блоки, сгруппированные по важности.
* **Функционал**: Прозрачность процессов, карточки с глубоким hover-эффектом и интеграция с CRM-лидами.

---

## 4. Стратегия разработки (MVP: 4 месяца)

### Roadmap

1. **Auth & Roles**: Реализация Spring Security + JWT. Разделение доступа: Client / Employee / Admin.
2. **Public Site**: Landing-визитка с Canvas-фоном, витрина курсов (защищенный доступ через MinIO presigned URL).
3. **CRM Core**: Модуль управления задачами, привязка клиентов к конкретным сотрудникам.
4. **Integration**: Адаптер для 1С (интерфейс `Integration1CPort` + `NoOp` реализация на старте).
5. **Analytics & Trust**: Интеграция калькулятора налоговой выгоды, блоков с кейсами и безопасностью.

---

## 5. Важные технические решения

### Row-level Security

В CRM-системе проверка доступа должна происходить на уровне бизнес-логики:

* *Ошибочно*: Проверка только роли пользователя.
* *Верно*: Проверка принадлежности записи (например, `task.owner_id == employee.id`) на уровне сервисного слоя бэкенда.

### Идемпотентность оплат

Все транзакции (включая QR-оплату) должны иметь ключ идемпотентности в БД. Повторные Webhook-запросы от платежного провайдера должны игнорироваться системой, если ключ уже обработан.

### Работа с данными (Type Safety)

Контракт между фронтендом и бэкендом гарантируется OpenAPI-спецификацией. Используем `openapi-typescript` для автогенерации типов, чтобы избежать ручного дублирования DTO.

---

## 6. Рекомендации по UX/UI

* **Типографика**: Использование жирного шрифта (`font-black`) в заголовках для "финансового" ощущения надежности.
* **Цветовая палитра**: `#00562D` (темно-зеленый — основной) и `#F5F5DC` (бежевый — фон).
* **Динамика**: Использование `leading-[0.9]` и `tracking-tighter` для создания премиального, плотного текста в заголовках.
