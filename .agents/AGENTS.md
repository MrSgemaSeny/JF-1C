# Project Rules

- **Docker**: The user does not care about Docker at this stage. Do not suggest, setup, or configure Docker (e.g. `Dockerfile`, `docker-compose.yml`, containerization) unless the user explicitly requests it. Focus on local development and native execution.

- **Communication**: NEVER use emojis in any responses, artifacts, or code. The user strictly forbids emojis.

- **Tests before pushing**: Never push to branches if it compared with errors.
- **Git Workflow**: Do not automatically commit and push small changes. Only git push if the changes are significant (e.g., 40+ lines of code modified) or if the user explicitly requests a push.
