# Zhan Finance Platform (JF-1C)
### Собственная бизнес-платформа вместо аренды чужой CRM

**Zhan Finance Platform** — это готовая веб-система, которая заменяет связку из Bitrix24, отдельного биллинга, отдельной техподдержки в мессенджерах и отдельной LMS для обучения сотрудников — одним продуктом, который принадлежит бизнесу целиком: с кодом, данными и возможностью развивать под себя без ограничений вендора.

Платформа уже реализована и работает: сайт компании, CRM с воронкой продаж, личные кабинеты клиентов и сотрудников, биллинг, документооборот, чат в реальном времени, система обучения и полный контроль доступа по ролям — всё в едином интерфейсе, на собственной инфраструктуре.

---

## Почему это ценно

Компании, которые растут, рано или поздно упираются в одну и ту же стену: универсальные SaaS-решения (Bitrix24, amoCRM и подобные) стоят дороже с каждым новым пользователем, ограничивают кастомизацию, хранят данные клиентов на чужих серверах и требуют оплаты за каждый «модуль», который на самом деле нужен бизнесу целиком.

**Zhan Finance Platform снимает все четыре ограничения одновременно:**

| Проблема аренды SaaS | Решение платформы |
|---|---|
| Растущая абонентская плата за пользователя | Единоразовая разработка, дальнейшая эксплуатация — на своих условиях |
| Данные клиентов на серверах третьей стороны | Собственная база данных (PostgreSQL), полный контроль над хранением |
| Типовой интерфейс «для всех», не под конкретный процесс | Система изначально спроектирована под бизнес-процессы бухгалтерского/финансового сервиса: воронка сделок, услуги, счета, документы — как единый рабочий поток |
| Разрозненные инструменты (CRM + чат + LMS + документооборот отдельно) | Всё объединено в одном продукте с единой авторизацией и единой базой пользователей |

Это не MVP и не прототип. Это работающая система с 218 backend-классами, 191 frontend-файлом, 43 миграциями базы данных, полноценным REST API из более чем 25 контроллеров, ролевой моделью на 4 типа пользователей и покрытием автотестами ключевых сценариев.

## Что умеет платформа

### 🎯 CRM и управление сделками
Настраиваемая воронка продаж с канбан-доской и drag-and-drop, гибкая фильтрация задач по любым критериям, подзадачи, комментарии, полная история изменений по каждой сделке, теги, привязка оказанных услуг к сделке, массовое обновление задач, экспорт данных для отчётности. Отдельный «пул задач» позволяет сотрудникам самостоятельно разбирать входящие заявки.

### 💰 Биллинг и подписки
Выставление счетов клиентам с автогенерацией PDF (включая корректную поддержку кириллицы), учёт разовых и регулярных платежей (подписок), привязка счетов к конкретным сделкам — руководитель в любой момент видит, какая сделка сколько принесла денег.

### 📄 Документооборот
Библиотека шаблонов документов, генерация документов по шаблону, безопасное хранение файлов (гибко переключается между локальной файловой системой и базой данных), контроль доступа к каждому документу на уровне отдельного файла.

### 🎓 Встроенная система обучения (LMS)
Курсы, разбитые на главы и уроки — видео, презентации, документы. Отдельная роль «ученик» с отслеживанием прогресса прохождения каждого урока. Готовый инструмент для онбординга новых сотрудников или для продажи образовательного продукта клиентам.

### 💬 Чат в реальном времени
Полноценный мессенджер на WebSocket с проверкой прав доступа не только на подключение, но и на каждую подписку и отправку сообщения — приватная переписка клиента с бухгалтером защищена на архитектурном уровне, а не только «на бумаге».

### 🔔 Уведомления в трёх каналах
Внутриплатформенные уведомления, email-рассылка через SMTP и автоматический ежедневный контроль дедлайнов по задачам — система сама напоминает сотрудникам и клиентам о горящих сроках, без ручного контроля.

### 🔍 Единый поиск по всей системе
Один поисковый запрос — результаты сразу по пользователям, задачам и учебным материалам.

### 🛡️ Полный аудит действий
Каждое изменение в бизнес-данных автоматически фиксируется в журнале аудита на уровне базы данных — кто, что и когда изменил. Для финансовой предметной области это не опция, а необходимость, и она уже встроена.

### 🌐 Публичный сайт с лидогенерацией
Лендинг с описанием услуг, тарифами, отзывами, командой, статистикой компании — и формой заявки с прикреплением файлов, которая напрямую попадает в очередь лидов внутри CRM. Ни один обращение с сайта не теряется.

### 🔐 Продуманная ролевая модель
Четыре роли — Администратор, Сотрудник, Клиент, Ученик — с полностью изолированными интерфейсами и правами доступа. Регистрация новых сотрудников проходит модерацию администратором. Каждая роль видит только свою часть системы.

### 🌍 Двуязычность из коробки
Русский и английский язык поддерживаются на всех уровнях — от интерфейса до справочных данных в базе (услуги, стадии воронки продаж). Платформа готова к выходу на международную аудиторию без дополнительной локализации.

## На чём построено

Платформа собрана на современном, проверенном временем и востребованном на рынке труда стеке — это не экзотика, а технологии, на которых строится большинство серьёзных бизнес-систем в мире:

**Backend:** Java 17, Spring Boot 4.1, Spring Security, Spring Data JPA, Spring WebSocket, PostgreSQL, Flyway, JWT-аутентификация, Google OAuth, Caffeine и Redis для кэширования, генерация PDF-документов, SMTP-рассылка.

**Frontend:** React 19, TypeScript, Vite 8, Tailwind CSS 4, TanStack Query, React Router, @dnd-kit для drag-and-drop интерфейсов, WebSocket-клиент для чата в реальном времени, i18next для мультиязычности.

**Архитектура:** backend разбит на 14 независимых бизнес-модулей (auth, CRM, billing, documents, courses, chat, notifications, audit, admin, search, services и другие), frontend построен по методологии Feature-Sliced Design — индустриальному стандарту организации крупных React-приложений. Такая архитектура означает, что систему легко расширять новыми модулями, не трогая существующие, и легко передать новой команде разработчиков — код организован по понятным, общепринятым правилам, а не «как получилось».

**Инфраструктура:** контейнеризация через Docker Compose (PostgreSQL + Redis), CI/CD на GitHub Actions, деплой бэкенда на Fly.io, фронтенда — через GitHub Pages с собственным доменом.

**Безопасность:** JWT access/refresh токены, BCrypt-хеширование паролей, rate limiting на аутентификации, полностью stateless-сессии, авторизация на уровне каждого WebSocket-канала, разграничение доступа к данным на уровне сервисов, защищённая и отключаемая в проде Swagger-документация API.

**Качество кода:** 12 тестовых классов на бэкенде (unit и интеграционные тесты ключевых сценариев — авторизация, задачи, воронка продаж, биллинг, заявки), тестовое покрытие ключевых компонентов на фронтенде, версионируемая через Flyway история изменений схемы базы данных из 43 миграций — вся эволюция продукта задокументирована и воспроизводима.

## Кому и зачем это подходит

- **Владельцу бухгалтерской/финансовой компании** — готовая цифровая инфраструктура для обслуживания клиентов, которую не нужно арендовывать и допиливать под себя годами.
- **Инвестору или покупателю технологического актива** — работающий продукт с реальными пользователями (интерфейс, тексты и тарифы — не демо-заглушки, а контент действующей компании), а не идея на салфетке.
- **Технической команде, которая возьмёт проект дальше** — понятная модульная архитектура, тесты, версионируемые миграции и современный стек, в котором не нужно разбираться неделями.
- **Бизнесу, который хочет отказаться от нескольких SaaS-подписок разом** — CRM, биллинг, документооборот, чат и LMS уже объединены в одном продукте.

## Итог

Zhan Finance Platform (JF-1C) — это не набросок идеи, а работающая, протестированная и уже используемая система полного цикла: от первого обращения клиента с сайта до выставленного счёта, подписанного документа и пройденного курса обучения. Платформа построена на современном стеке, имеет чёткую модульную архитектуру и готова как к дальнейшему развитию, так и к передаче новой команде — без скрытых долгов и без зависимости от чужой инфраструктуры.
