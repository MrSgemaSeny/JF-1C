# Реализация сайта-визитки (Landing Page) на архитектуре FSD

Данный документ описывает структуру и организацию кода для реализации сайта-визитки бухгалтерской компании в рамках методологии Feature-Sliced Design (FSD).

## 1. Структура слоев

### `shared` (Общий уровень)
*   `api/`: Базовый экземпляр Axios, конфигурация API, типы, сгенерированные из OpenAPI.
*   `ui/`: Общие компоненты (Button, Input, Card, Logo, Container), используемые во всем проекте.
*   `lib/`: Общие функции-утилиты (валидаторы ИИН, форматирование дат).

### `entities` (Сущности)
*   `service/`: Модель услуги (интерфейс `Service`, функции получения списка услуг).
*   `employee/`: Модель сотрудника (интерфейс `Employee`).
*   `company/`: Информация о компании (описание, контакты).

### `features` (Бизнес-логика)
*   `send-request/`: Форма заказа услуги / первичной консультации (валидация полей, API-запрос).
*   `auth-link/`: Компонент кнопки входа, перенаправляющий на ЛК.

### `widgets` (Блоки)
*   `header/`: Навигация (ссылки на разделы, кнопка входа).
*   `services-list/`: Сетка карточек с описанием услуг (использует `entities/service`).
*   `hero/`: Заголовок, оффер, призыв к действию.
*   `footer/`: Контактная информация, копирайты.

### `pages` (Страницы)
*   `home/`: Сборка страницы из виджетов (`Header` + `Hero` + `ServicesList` + `Footer`).

## 2. Правила реализации (Constraints)

1.  **Строгость импортов**:
    *   `pages` импортирует из `widgets`, `features`, `entities`, `shared`.
    *   `widgets` импортирует из `features`, `entities`, `shared`.
    *   `features` импортирует из `entities`, `shared`.
    *   `entities` импортирует из `shared`.
    *   Запрещены импорты "снизу-вверх" (например, `entities` не может импортировать `features`).
2.  **Barrel-файлы**: Каждый уровень папки обязан содержать `index.ts`, который экспортирует только публичный API этого слоя/модуля.
3.  **Контракты**: Для взаимодействия с бэкендом (Spring Boot) используем автогенерацию типов. Типы данных в `entities` должны соответствовать DTO, полученным из OpenAPI-спецификации.

## 3. Технологический стек реализации
*   **Framework**: React 18, Vite.
*   **Язык**: TypeScript (строгая типизация).
*   **Стилизация**: Tailwind CSS (согласно фирменному стилю: основной зелёный `#00562D`, бежевый `#F5F5DC`).
*   **Анимации**: HTML5 Canvas (для динамического фонового слоя).
