# Zhan Finance (JF-1C)

Zhan Finance (JF-1C) — это внутренний веб-проект для цифровизации работы бухгалтерской и финансовой компании. Проект задумана как более специализированная и управляемая альтернатива универсальным CRM/порталам, таким как Bitrix24: он объединяет работу с клиентами, задачами, документами, чатами, уведомлениями и billing-процессами в одной системе.

## Что это за проект

JF-1C — это не просто CRM. Это попытка собрать собственную платформу для:

- ведения клиентских задач и бизнес-процессов;
- работы бухгалтеров, менеджеров и администраторов в едином интерфейсе;
- обмена документами и сообщениями с клиентами;
- управления счетами, заявками и сервисными сценариями;
- постепенного перехода от типовых решений к собственной, гибкой системе.

Проект особенно ориентирован на бизнес-модель бухгалтерского и финансового обслуживания, где важны:

- прозрачность процессов;
- контроль сроков и статусов;
- разграничение прав доступа;
- работа с документами и коммуникацией;
- возможность кастомизации под конкретный бизнес.

## Главная цель

Сделать удобную и безопасную платформу, которая помогает компании:

- управлять клиентскими задачами и внутренними процессами;
- ускорять коммуникацию между сотрудниками и клиентами;
- хранить и распространять документы в контролируемом формате;
- строить собственную инфраструктуру вместо зависимости от универсальных SaaS-решений.

## Смысл проекта

Идея JF-1C — перейти от «универсального коробочного портала» к собственной технологической платформе, которая лучше соответствует специфике бухгалтерского и финансового сервиса. Вместо перегруженного интерфейса и жестких шаблонов система стремится быть:

- более понятной для сотрудников;
- более адаптируемой под реальные бизнес-процессы;
- более контролируемой с точки зрения безопасности и данных;
- более независимой от внешних платформ.

## Основные возможности

### Бэкенд
- аутентификация и авторизация;
- JWT access/refresh tokens;
- Google OAuth;
- роли: ADMIN, EMPLOYEE, CLIENT, LEARNER;
- CRM-модуль: задачи, комментарии, активности, календари;
- документы: загрузка, хранение и доступ;
- уведомления и чаты;
- billing-модуль для счетов и подписок;
- курсы и учебный контент;
- поиск по данным системы.

### Фронтенд
- SPA на React;
- роли и отдельные dashboard-страницы;
- задачи и канбан-подобные рабочие процессы;
- клиентские и административные сценарии;
- система авторизации и личных кабинетов;
- интерфейсы для чатов, документов, календаря, сервисов и контента.

## Технологический стек

### Backend
- Java 17
- Spring Boot 4.1
- Spring Security
- Spring Data JPA
- Spring WebSocket
- Flyway
- PostgreSQL
- JWT
- OpenAPI / Swagger
- Mail, validation, file storage support

### Frontend
- React 19
- TypeScript
- Vite 8
- Tailwind CSS 4
- TanStack Query
- React Router
- @dnd-kit для интерфейсов перетаскивания
- WebSocket клиент для чата

### Инфраструктура
- Docker Compose
- Postgres + Redis
- Fly.io / GitHub Pages deployment-oriented setup

## Структура репозитория

- zhan-finance-backend — серверная часть на Java/Spring Boot
- zhan-finance-frontend — клиентская часть на React/TypeScript
- docker-compose.yml — локальный запуск инфраструктуры
- markdowns_obsidian — аналитика, планы, документы по проекту

## Запуск локально

### Требования
- Docker и Docker Compose
- Java 17+
- Node.js 20+

### Через Docker

```bash
docker compose up --build
```

### Backend

```bash
cd zhan-finance-backend
./gradlew bootRun
```

### Frontend

```bash
cd zhan-finance-frontend
npm install
npm run dev
```

## Текущее состояние

Проект находится в активной разработке и уже содержит базовую реализацию ключевых модулей, включая:

- авторизацию и роли;
- CRM-задачи и клиентские workflows;
- документы и уведомления;
- чат и календарь;
- курсы и базовые интерфейсы.

При этом ряд функций всё ещё находится в стадии доработки или расширения, включая безопасность, платежи, аудит действий и дальнейшую зрелость бизнес-логики.

## Почему это важно

JF-1C — это попытка создать не просто «ещё один CRM», а собственную технологическую основу для бизнеса, который работает с клиентами, финансами, документами и операционными процессами ежедневно. Проект важен тем, что он объединяет:

- продуктовую логику;
- внутренние процессы;
- клиентский опыт;
- техническую независимость.

## Кратко

JF-1C — это кастомная платформа для бухгалтерского и финансового сервиса, которая строится как современное веб-приложение с понятной архитектурой, ролями, задачами, документами и коммуникацией. Его цель — дать бизнесу инструмент, который лучше отражает реальные процессы, чем стандартные универсальные решения.
